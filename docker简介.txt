docker image
容器的基石（好比容器的源代码，存放启动的文件，容器基于镜像启动）
层叠的只读文件系统(bootfs引导文件系统,rootfs只读文件系统，后面会有更多的只读文件系统，
读写文件系统)
联合加载（union mount）（一次加载多个只读文件系统）
docker container
通过镜像启动
启动和执行阶段（启动会加载一个读写文件系统，在上面的文件系统最上层）
写时复制
docker registr （仓库）
公有，私有
docker hub 提供各种镜像
https://www.docker.com/tryit/(在线演示地址)
功能：
文件系统隔离：每个容器都有自己的root文件系统
进程隔离：都运行在自己的进程环境中
网络隔离： 容器简的虚拟网络接口和ip地址都是分开的
资源隔离和分组： 使用cgp将cpu和内存之类的资源独立分配给每个docker容器
